# PRD-001 — MGR Foundation

**Project:** Markdown Game Runtime (MGR)

**Version:** 1.0

**Status:** Draft

---

# 1. Mục tiêu

PRD này định nghĩa nền tảng đầu tiên của MGR.

Mục tiêu không phải tạo game.

Mục tiêu không phải tạo gameplay.

Mục tiêu là xây dựng một Prompt Programming Framework có thể mở rộng trong nhiều năm.

Đây là PRD quan trọng nhất của toàn bộ dự án.

Sau khi Foundation hoàn thành, mọi tính năng khác phải được xây dựng dựa trên nền tảng này.

---

# 2. Mục tiêu sản phẩm

MGR là một compiler dành cho Prompt Programming.

Developer viết nhiều file Markdown.

MGR biên dịch chúng thành một Prompt Specification duy nhất để một LLM (ví dụ ChatGPT) đóng vai Runtime và thực thi trò chơi theo lượt.

MGR không chạy game.

MGR không mô phỏng game.

MGR không chứa AI.

LLM là Runtime.

Markdown là Source Code.

MGR là Compiler.

---

# 3. Phạm vi

Phiên bản Foundation chỉ giải quyết các vấn đề sau:

* Chuẩn cấu trúc project
* Đọc source Markdown
* Phân tích project
* Build Prompt Specification
* Validation
* CLI

Không xây gameplay.

Không xây plugin.

Không xây template game.

Không xây thư viện luật chơi.

# 4. Nguyên tắc thiết kế

## 4.1 Markdown First

Markdown là ngôn ngữ duy nhất của project.

Không yêu cầu DSL riêng.

Không yêu cầu JSON để mô tả gameplay.

Markdown phải vừa là tài liệu, vừa là source code.

---

## 4.2 Human Readable

Mọi file đều có thể đọc như tài liệu kỹ thuật.

Không được sinh ra cú pháp khó đọc.

---

## 4.3 Predictable

Build cùng một source phải luôn sinh cùng một output.

Không có hành vi ngẫu nhiên trong compiler.

---

## 4.4 Extensible

Mọi thành phần phải có khả năng mở rộng mà không phá vỡ project cũ.

---

## 4.5 Engine Agnostic

Foundation không phụ thuộc ChatGPT.

Output phải có thể dùng với bất kỳ LLM nào hỗ trợ prompt dài.

# 6. Thành phần Core

Foundation bao gồm năm thành phần.

## Parser

Đọc project.

Đọc Markdown.

Nhận biết Directive.

Xây dựng AST.

---

## Project Graph

Xây dựng đồ thị phụ thuộc.

Theo dõi import.

Theo dõi section.

Theo dõi module.

Phát hiện vòng lặp.

---

## Validator

Kiểm tra tính hợp lệ của project.

Ví dụ:

* thiếu file
* import sai
* section trùng
* directive sai
* dependency cycle

Validator không sửa lỗi.

Validator chỉ báo lỗi.

---

## Bundler

Ghép project thành Prompt Specification.

Bundler quyết định thứ tự module.

Bundler xử lý include.

Bundler xử lý import.

---

## Optimizer

Loại bỏ phần dư thừa.

Chuẩn hóa heading.

Chuẩn hóa khoảng trắng.

Chuẩn hóa cấu trúc.

Không thay đổi nội dung logic.

# 7. CLI

Foundation phải cung cấp CLI.

Các lệnh tối thiểu:

```text
mgr init

mgr build

mgr validate

mgr doctor
```

CLI phải hoạt động trên Windows, macOS và Linux.

---

# 8. Cấu trúc project chuẩn

```text
project/

    mgr.config.json

    src/

    dist/
```

MGR không ép buộc cấu trúc bên trong `src`.

Foundation chỉ yêu cầu mọi file nguồn đều nằm trong `src`.

Các chuẩn thư mục cao hơn sẽ được định nghĩa ở PRD khác.

---

# 9. Prompt Specification

Bundler sinh đúng một file.

Ví dụ:

```text
dist/

    game.md
```

Đây là artifact duy nhất mà người dùng đưa vào LLM.

Foundation không quan tâm game gì.

Foundation chỉ quan tâm Prompt Specification hợp lệ.

---

# 10. Directive

Foundation hỗ trợ Directive mức tối thiểu.

Bắt buộc:

* `@import`
* `@section`

Directive khác sẽ được bổ sung sau.

# 15. Tiêu chí hoàn thành

Foundation được coi là hoàn thành khi:

* Có thể khởi tạo project mới.
* Có thể đọc project Markdown.
* Có thể tạo Project Graph.
* Có thể kiểm tra lỗi.
* Có thể build thành một Prompt Specification.
* Có thể sinh ra cùng một output từ cùng một source.
* Có thể mở rộng bằng các PRD tiếp theo mà không cần thay đổi kiến trúc nền.

Foundation không cần biết bất kỳ luật chơi hay thể loại game nào. Nó chỉ chịu trách nhiệm cung cấp hạ tầng ổn định để mọi hệ thống phía trên có thể được xây dựng trong tương lai.
